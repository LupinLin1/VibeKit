# VibeKit — Vibe Coding 硬件加速套件

**VibeKit** 是一套专为 **Vibe Coding** 场景设计的 macOS 工具，核心目标是让你在和 Claude Code 协作编程时，双手尽量不离开麦克风和思考，而不是键盘。

---

## 什么是 Vibe Coding？

Vibe Coding 是一种以自然语言为主要输入、AI 为主要执行者的编程方式。你描述意图，AI 写代码；你审阅、确认、调整方向，AI 继续迭代。键盘的角色从"打字工具"变成"导航工具"。

VibeKit 的设计理念就是让这个"导航"动作尽可能轻量。

---

## 硬件套件

完整的 VibeKit 体验由三部分组成：

### 1. 收音麦
用于语音输入。推荐桌面指向性麦克风，减少环境噪音干扰。语音是主要的指令输入方式。

### 2. 豆包输入法（语音转文字）
字节跳动出品的输入法，内置高精度实时语音识别。在 Claude Code 的输入框里直接按住说话，松开即转为文字，无需额外软件。


### 3. JX-11 蓝牙遥控器 + VibeKit 守护进程（本项目）
一个 5 键 BLE HID 遥控器，配合本项目的守护进程，将物理按键映射为 Claude Code 常用操作。这样你可以在说话的同时，用另一只手操作遥控器完成确认、取消、切换等动作，不需要伸手去够键盘。

## 典型工作流

```
你（说话）→ 豆包输入法 → Claude Code 输入框
                                    ↓
               Claude Code 思考、写代码、提问
                                    ↓
你（JX-11）→ ↩ 确认 / ESC 取消 / ⌘⇥ 切换应用
```

不需要低头看键盘，不需要移动鼠标找按钮，整个交互流程保持在"说话 + 拇指"这个最小动作范围内。

默认映射针对 Claude Code 使用场景优化：

| 按键 | 单击 | 双击 | 长按 |

| 滚轮下 | ↓ 下移光标 | — | — |

## 安装

### 系统要求

- macOS 12+
- Python 3.9+
- JX-11 蓝牙遥控器（VID `0x05AC` / PID `0x0220`）

### 步骤

1. 将 `JX-11.app` 拖入 `/Applications/`
2. 双击启动，按引导完成依赖安装和权限授权
3. 连接 JX-11 设备（蓝牙配对）
4. 守护进程自动注册为登录项，开机自启

### 所需权限

| 权限 | 用途 | 开启位置 |
|------|------|----------|
| 辅助功能 | 模拟键盘输入 | 系统设置 → 隐私与安全性 → 辅助功能 |
| 输入监控 | 读取 HID 设备数据 | 系统设置 → 隐私与安全性 → 输入监控 |

## 配置自定义

双击 `JX-11.app`（守护进程运行时）打开配置界面，可为每个按键设置：

- **普通按键**：任意键码 + 修饰键组合（⌘ ⌃ ⌥ ⇧）
- **双修饰键**：连续发送两次修饰键（如 ⌥⌥ 触发 Raycast）— 兼容旧配置
- **切换输入法**：通过 TIS API 直接循环切换输入法
- **轮换窗口**：按字母顺序依次激活所有前台应用

配置实时生效，无需重启守护进程。配置保存在 `~/.config/jx11/config.json`。

## 技术说明

### 项目结构

```
JX-11.app/
├── Contents/
│   ├── MacOS/
│   │   └── JX-11          # Bash 启动脚本
│   └── Resources/
│       ├── daemon.py       # 守护进程主体
│       ├── config_ui.py    # 配置界面（tkinter）
│       ├── setup.py        # 首次引导安装
│       └── switch_input_method.swift  # 输入法切换（TIS API）
```

### 设备通信

JX-11 通过 BLE 模拟为 HID 设备接入 macOS。守护进程使用 `hidapi` 以非阻塞模式轮询设备（1ms 间隔）：

- `data[1]`：按键状态，`0x07` = 按下，其他值 = 松开
- `data[2:5]`：按键指纹，唯一标识每个物理按键（松开时归零，因此需要在按下时记录）

### 多击检测逻辑

- **双击**：松开后启动计时器（0.55s），若在窗口内再次按下则触发双击动作，取消单击计时器
- **长按**：记录按下时间戳，松开时计算持续时长，超过阈值（0.50s）触发长按
- **抑制逻辑**：双击触发后，第二次松开不再启动单击计时器，避免多余触发

### 键盘模拟

- 普通按键：通过 `osascript` 调用 System Events 的 `key code` 指令
- 输入法切换：通过 Swift 调用 Carbon TIS（Text Input Source）API 直接切换输入源，比模拟按键更可靠

### 日志

```
~/Library/Logs/jx11-daemon.log
```

## 开发者指南

### 迭代修改

直接编辑 `JX-11.app/Contents/Resources/` 下的 Python 文件，改完后重启守护进程生效：

```bash
# 重启守护进程
pkill -f daemon.py && open JX-11.app

# 实时查看日志
tail -f ~/Library/Logs/jx11-daemon.log

# 确认进程状态
pgrep -f daemon.py
```

> 配置文件（`~/.config/jx11/config.json`）改动**无需重启**，下次按键时自动读取。

### 运行测试

```bash
pip install pytest
pytest tests/
```

### 安装开发依赖

```bash
pip install --break-system-packages -r requirements.txt
```
